# Contribution Types

## Overview

Proof of Contribution requires a broad taxonomy of contribution types. A blockchain ecosystem is not sustained only by code commits or token transactions. Documentation, research, education, infrastructure, governance, and community support may all contribute to long-term ecosystem quality.

## Contribution Categories

| Category | Description | Possible Evidence |
|---|---|---|
| Code Contribution | Protocol, tooling, scripts, examples, or application code | Pull requests, commits, releases |
| Documentation | Guides, references, FAQs, configuration notes, tutorials | Documentation commits, published pages |
| Issue Reporting | Bug reports, feature requests, unclear documentation reports | GitHub issues, maintainer responses |
| Infrastructure Operation | Running nodes, RPC services, monitoring tools, uptime support | Node records, dashboards, logs, attestations |
| Governance Participation | Voting, proposal writing, proposal analysis, governance discussion | Governance records, forum posts, proposals |
| Research | Technical papers, comparative studies, threat models, evaluation matrices | Research repositories, papers, citations |
| Education | User guides, explainers, onboarding sessions, community help | Articles, videos, forum answers |
| Translation and Localization | Translating technical or educational content | Published translations, review records |
| Security Contribution | Responsible disclosure, audit notes, vulnerability analysis | Security reports, maintainer acknowledgments |
| Community Coordination | Moderation, event organization, contributor support | Moderator attestations, event records |

## Quality Considerations

Contribution quality should not be measured only by quantity. A single high-impact security report may be more valuable than many low-effort posts. A short documentation correction may meaningfully improve onboarding if it resolves a common blocker.

Quality indicators may include:

- Accuracy
- Originality
- Usefulness
- Review acceptance
- Long-term relevance
- Evidence quality
- Maintenance effort
- Community impact

## Contribution Boundaries

The system should avoid counting every action as meaningful contribution. Examples of weak or ambiguous evidence include:

- Automated engagement
- Duplicate posts
- Low-effort comments
- Unverified screenshots
- Claims without evidence
- Content copied without attribution
- Activity created only for task completion

## Research Position

A contribution taxonomy should help communities preserve useful work while resisting shallow activity farming. The framework should remain flexible enough to recognize different types of human contribution without pretending that all contribution types are directly comparable.
